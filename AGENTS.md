# AI 通用指令入口

本文件是 `study` 仓库的统一 AI / Agent 入口。下面的路径均以仓库根目录为基准。处理 `apps/` 中的项目时，先完整读取这里引用的通用规则，再读取沿目录层级出现的项目专属 `AGENTS.md`。

## 读取顺序

1. `.ai/readme.md`
   - 理解通用规则的分类方式，以及通用规则与项目规则的边界。

2. `.ai/workflows/Karpathy.md`
   - 遵守先澄清、简单优先、最小范围改动和目标驱动执行的工作方式。

3. `.ai/standards/code-style.md`
   - 遵守少抽象、逻辑就近、最小修改和项目一致性的编码规范。

4. `apps/AGENTS.md`、当前项目目录中的 `AGENTS.md` 与其引用的专属规则
   - 框架、语言和项目规则用于补充本文件；出现冲突时，不得放宽上级通用约束。

## 通用执行约束

- 如果用户没有明确要求，不主动新增复杂架构或通用抽象。
- 修改前先确认目标、影响范围和成功标准。
- 只改和当前任务直接相关的文件，并保留项目已有风格。
- 使用与风险相称且被项目规则允许的方式验证改动；无法执行验证时明确说明。
- 子项目新增长期规则时，将技术栈无关的编码思想放回仓根 `.ai/`，将框架或项目专属规则留在子项目内。
