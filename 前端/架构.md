# 模块联邦

## 路由
- 和菜单定相同层级的 path，方便做菜单高亮
  - 一级 /task
  - 二级 /task/myTask
  - 多个相同路由，不同菜单，带参数 /task/myTask?key=val

### 子app
```js
// 提供 routes 配置
[
  {
    path: '/path',
    name: 'name',
    component: () => import('xxx')
  }
]
```

### 主app
```js
// 把子路由改造成
[
  {
    path: '/app1/path',
    name: 'app1-name',
    component: () => import('xxx')
  }
]
```

## ajax

### 子app
```js
// 提供配置
{
  // 主app中要在 apiList 每个path前增加 /app1 这类前缀
  apiList: { getUsrs: '/api/getUsrs' },
  ajaxList: { getUsrs() { ajax.post('getUsrs') } }
}
```

### 主app
```js
// 组装全局对象
this.$ajax = {
  // 子设计成方法，方便内部懒加载子app资源后再调用
  app1(method, params, config) {}, 
  app2(method, params, config) {},
  getUsrs() {}
}

// 调用主app
this.$ajax.getUsrs()

// 调用子app，做成方法方便做懒加载
this.$ajax.app1(method, params, config)
```

## 组件

## 主app
```js
// 主组件
this.$modal.show()

// 子组件，做成方法方便做懒加载
this.$app1('组件名', method, ...args)
this.$app2('组件名', method, ...args)
```

# Monorepo（单一代码仓库）

Monorepo 就是把多个相关项目放在同一个 Git 仓库里管理。例如：后台系统、移动端、组件库、工具库都在一个仓库下，而不是每个项目各自一个仓库。

简单说，就是“一个仓库管理多个项目”，而不是“一个项目一个仓库”。

它的用处：

- 让多个项目之间共享代码更方便，公共组件库和工具库可以直接引用本地代码。
- 所有项目共用同一个依赖锁文件，依赖版本更统一。
- 如果要改一个公共库，跨项目修改可以一次性提交，重构更容易。
- CI/CD 可以根据改动只构建受影响的项目，效率更高。

常见技术网站（按热门顺序）：

- Nx: https://nx.dev
- Turborepo: https://turbo.build/repo
- pnpm Workspaces: https://pnpm.io/workspaces
