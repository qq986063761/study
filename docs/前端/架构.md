# 架构

## 模块联邦

### 路由

- 和菜单定相同层级的 path，方便做菜单高亮
  - 一级 /task
  - 二级 /task/myTask
  - 多个相同路由，不同菜单，带参数 /task/myTask?key=val

#### 子app

```js
// 提供 routes 配置
[
  {
    path: '/path',
    name: 'name',
    component: () => import('xxx')
  }
]
```

#### 全局调用

```js
// 主应用加载子应用路由后，统一补上前缀
[
  {
    path: '/app1/path',
    name: 'app1-name',
    component: () => import('xxx')
  }
]

// 跳主应用路由
this.$router.push({name: 'home'})

// 跳子应用路由
this.$router.push({name: 'app1-home'})
```

### ajax

#### 子app

```js
// 提供配置
{
  // 主app中要在 apiList 每个path前增加 /app1 这类前缀
  apiList: { getUsrs: '/api/getUsrs' },
  ajaxList: { getUsrs() { ajax.post('getUsrs') } }
}
```

#### 全局调用

```js
// 封装全局对象
Vue.prototype.$ajax = {
  // 子设计成方法，方便内部懒加载子app资源后再调用
  app1(method, params, config) {}, 
  app2(method, params, config) {},
  getUsrs() {}
}

// 调用主app接口
this.$ajax.getUsrs()

// 调用子app接口，做成方法方便做懒加载
this.$ajax.app1(method, params, config)
this.$ajax.app2(method, params, config)
```

### 组件

#### 子app

```js
// 暴露全局组件
export default {
  modal: Modal
}

// 暴露局部组件
export default {
  'item': () => import('./item')
}
```

#### 全局调用

```js
// 调用主全局组件
this.$modal.show()

// 调用子全局组件，做成方法方便做懒加载
this.$app1('组件名', method, ...args)
this.$app2('组件名', method, ...args)

// 局部组件直接使用
<child-item></child-item>
```

### vue2 vuex store

#### 子app

```js
// 暴露普通 Vuex module，不写 namespaced，主app中会补上命名空间
export default {
  state: {},
  getters: {},
  mutations: {},
  actions: {}
}
```

#### 全局调用

```js
// 主app加载子app store 后，统一加命名空间注册
store.registerModule('app1', {
  ...app1StoreModule,
  namespaced: true
})

// 调用主app
this.$store.state.appName
this.$store.getters['doubleCount']
this.$store.commit('increment')
this.$store.dispatch('incrementAsync')

// 调用子app
this.$store.state.app1.count
this.$store.getters['app1/doubleCount']
this.$store.commit('app1/increment')
this.$store.dispatch('app1/incrementAsync')
```

### vue3 pinia store

#### 子app

```js
// 暴露store方法
import { defineStore } from 'pinia'
export const useStore = defineStore('app1', () => {
  
  return {
    add
  }
})
```

#### 全局调用

```js
// 调用主app
this.$store.useStore()

// 调用子app
this.$store.app1('useStore', params)
this.$store.app2('useStore', params)
```

## Monorepo（单一代码仓库）

Monorepo 就是把多个相关项目放在同一个 Git 仓库里管理。例如：后台系统、移动端、组件库、工具库都在一个仓库下，而不是每个项目各自一个仓库。

简单说，就是“一个仓库管理多个项目”，而不是“一个项目一个仓库”。

它的用处：

- 让多个项目之间共享代码更方便，公共组件库和工具库可以直接引用本地代码。
- 所有项目共用同一个依赖锁文件，依赖版本更统一。
- 如果要改一个公共库，跨项目修改可以一次性提交，重构更容易。
- CI/CD 可以根据改动只构建受影响的项目，效率更高。

解决问题：

- 不同技术栈统一管理
- 多个仓库合并到一个仓库统一管理
- 公用工具库统一管理，方便共享
- 统一打包配置

常见技术网站（按热门顺序）：

- Nx: https://nx.dev（解决「大型团队工程化」的问题）
- Turborepo: https://turbo.build/repo（解决「构建速度」的问题）
- pnpm Workspaces: https://pnpm.io/workspaces（解决「项目组织」的问题（基础））
