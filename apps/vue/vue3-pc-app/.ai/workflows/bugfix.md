# Bugfix 工作流

修复缺陷时使用这个工作流。

## 检查清单

1. 复现问题，或明确识别失败行为。
2. 搜索相关代码路径和已有测试。
3. 用最小改动修复根因。
4. 可行时补充或更新回归测试。
5. 运行最窄但有效的验证命令。
6. 如果这个 bug 暴露了容易复发的坑，记录到 `.ai/memory/pitfalls.md`。

## 调试建议

- 导航问题同时检查 route、菜单和组件状态。
- DOM 或 Element Plus 实例问题重点检查 ref 空值和生命周期时机。
- 响应式异常重点检查 props、reactive 对象或 store 的解构方式。
- 使用 router、Pinia 或 Element Plus 的组件测试，需要检查测试插件注入。

## 修复规则

- 不要用宽泛的 `try/catch` 掩盖错误。
- 不要为局部 bug 引入全局状态，除非确实需要共享。
- 不要为了局部问题重写整个组件。
- 除非 bug 本身要求改变，否则保持用户可见行为稳定。

## 文件作用

用这个文件回答：“如何修复 bug，同时不让周边代码变得更难信任？”
