# 重构工作流

在不打算改变行为、只调整结构时使用这个工作流。

## 检查清单

1. 明确哪些行为必须保持不变。
2. 阅读已有测试，识别测试缺口。
3. 小步移动或重命名代码。
4. 除非任务明确要求，否则保持公开契约稳定。
5. 运行类型检查和相关测试。
6. 重要架构决策记录到 `.ai/memory/decisions/`。

## 适合重构的目标

- 重复的组件逻辑，可以抽为 composable。
- 过大的页面区域，可以拆成聚焦组件。
- 混合了无关职责的 store。
- 路由和菜单定义开始不同步。

## 重构规则

- 不要把行为变化混进结构清理。
- 不要随意重命名 routes、stores、emitted events 或 props。
- 不要跨无关文件做格式化。
- 保留演示内容，除非任务要求删除或重设计。

## 验证建议

重构时优先运行：

```sh
npm run type-check
npm run test:unit
npm run build
```

如果影响路由、应用壳层或用户流程，再运行 Playwright。
