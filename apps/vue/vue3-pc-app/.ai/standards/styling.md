# 样式规范

这个文件定义 Vue 应用的样式编写预期。

## 默认规则

- Vue 组件样式默认使用 `scoped`，除非明确需要全局样式。
- 全局样式放在 `src/assets/css/base.css` 或其他命名清晰的全局样式文件。
- 当嵌套选择器或变量能提升可读性时，可以使用 SCSS。
- 优先使用 class 选择器，避免深度覆盖 Element Plus 内部结构。

## 布局

- 保持全高应用壳层：
  - `html`、`body` 和 `#app` 填满视口。
  - `#app` 使用 flex 布局。
- 页面布局要有稳定的间距、宽度和溢出行为。
- 避免 hover 状态、动态标签或加载文案导致布局抖动。

## Element Plus

- 优先使用 Element Plus 的组件属性和设计变量，再写自定义 CSS。
- 覆盖 Element Plus 样式时，把影响范围限制在当前页面或组件。
- 常见操作和菜单图标优先使用 `@element-plus/icons-vue`。

## 视觉风格

- PC 管理/演示风格应实用、清晰、可读。
- 避免单一色系铺满界面和厚重渐变。
- 避免卡片套卡片，也不要把整段页面区域做成漂浮卡片。
- 圆角保持克制，除非是在贴合 Element Plus 默认风格。

## 响应式

- 修改应用壳层、侧边栏、表格、表单或高密度控件时，需要关注窄屏。
- 文字空间不足时，优先换行或调整布局，避免重叠。
- 不要使用基于视口宽度的字体缩放。
- 除非有明确设计原因，否则保持默认字间距。

## 文件作用

用这个文件回答：“CSS 应该怎么写，才能让应用保持稳定和可读？”
