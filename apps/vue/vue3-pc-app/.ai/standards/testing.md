# 测试规范

这个文件定义 AI 助手在本仓库中处理测试的方式。

## 测试栈

- 单元测试：Vitest 和 Vue Test Utils。
- 浏览器测试：Playwright，位于 `e2e`。
- 类型验证：通过 `npm run type-check` 或 `npm run build` 执行 `vue-tsc`。

## 什么时候新增或更新测试

- composable、store 和有实际行为的组件应补单元测试。
- 路由级流程、导航、表单和依赖真实浏览器的交互应补 E2E 测试。
- 可见文案、路由或应用壳层行为变化时，要同步更新已有测试。
- 不要为用户不可观察的实现细节添加脆弱测试。

## 单元测试建议

- 挂载组件时只注入必要插件。
- 网络请求和浏览器 API 在边界处 mock。
- 关注事件、渲染状态和用户交互。
- 测试要可读，围绕行为而不是实现细节。

## E2E 测试建议

- 优先使用语义化 locator。
- 优先使用稳定可见文本，而不是 CSS 类选择器。
- 测试之间保持独立、确定。
- 新增侧边栏路由时，可以考虑补一个导航 E2E 测试。

## 验证命令

```sh
npm run type-check
npm run test:unit
npm run test:e2e
npm run build
```

纯文档改动通常不需要跑测试。涉及路由或应用壳层时，可行的话至少运行 `npm run build`。

## 文件作用

用这个文件回答：“这种改动应该做哪些验证？”
