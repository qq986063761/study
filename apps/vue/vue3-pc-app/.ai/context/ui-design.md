# UI 设计上下文

这个文件记录 AI 生成 UI 时应遵循的视觉和交互预期。

## 产品气质

这是一个面向 PC 的 Vue 3 应用，当前更接近管理后台或学习演示壳层。界面应该清晰、实用、易扫描，而不是营销站风格。

## 现有壳层

- 左侧侧边栏使用 Element Plus 菜单。
- 侧边栏可折叠：展开约 `200px`，折叠约 `64px`。
- 主内容区显示在 Element Plus scrollbar 内。
- 侧边栏当前为深蓝灰色，激活态使用 Element Plus 蓝色。

## UI 约定

- 表单、表格、菜单、弹窗、提示、标签页、按钮、空状态和反馈优先使用 Element Plus。
- 菜单项和图标按钮优先使用 `@element-plus/icons-vue`。
- 管理后台页面要保持信息密度可读：标题清晰、控件对齐、间距稳定、状态可见。
- 避免装饰性 hero、大号营销文案、漂浮式整段卡片和纯装饰渐变。
- 卡片只用于重复项、局部面板或需要明确边界的工具区域。

## 响应式预期

- 应用 PC 优先，但窄屏不应破版。
- 保持侧边栏折叠行为可用。
- 避免按钮、菜单项、表格、卡片和表单标签文字溢出。
- 优先通过换行或布局调整解决空间不足，不要用视口单位缩放字体。

## 交互规则

- 异步 UI 应提供明确的加载、空状态和错误状态。
- 按钮需要清楚表达动作，常见动作可以搭配图标。
- 路由导航要可预测：点击侧边栏菜单后跳转页面，并重置主内容滚动位置。
- 不要把关键操作隐藏成只有读说明才知道的行为。

## 文件作用

用这个文件回答：“新增页面或组件在这个应用里应该是什么感觉？”
