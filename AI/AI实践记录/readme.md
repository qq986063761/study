# ai 优点
- 逻辑布局上非常厉害，适合利用它写组件，业务功能
- 问题分析能力优秀，适合引导我们找出代码上的问题

# ai 不足
- C端设计高宝，还原度在细节上还不够

# 省 token 经验
- 模型不必要一直用顶级，日常场景使用当前主流模型即可，比如 sonnet
- 经常提问的内容，可以提取到 rules 不用反复提问，节省上下文 token
- 写 .claudeignore、.cursorignore 文件，忽略一些不必要的文件，节省上下文 token
- 常用命令做成 command 类似 /fix-i18n 这种
- 提问时，别每次都全量问，根据需求和bug的涉及范围，比如具体文件夹，或具体文件，给ai明确范围节省上下文
- 大体量需求，别一次提问，根据需求的功能点分多次提问，每次实现一个功能也能让ai实现更精确一些
- 对有 tab 功能的 ide，小范围需求直接用 tab 连续补全就行，用不到 chat 提问
- 相关功能完成后，即时清理对话，避免过期对话导致上下文过多
